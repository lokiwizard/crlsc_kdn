# CRLSC-KDN 具体实现方案

## 1. 目标

我要实现的是一个边云协同的无标签表征学习框架：

- 云端用 CLIP 对大规模数据集离线编码，建立语义特征库。
- 本地用 ResNet-18 在小规模无标签数据上做自监督学习。
- 训练本地模型时，用本地特征去云端特征库检索 top-k 邻居。
- 将邻居特征加权聚合成语义原型，约束本地特征向该原型对齐。

核心目标是验证一件事：

**在边端无标签、小样本条件下，引入云端 CLIP 语义原型后，本地轻量模型的表征质量是否优于单独做自监督学习。**

## 2. 我准备实现的最小可运行版本

第一版只实现最核心闭环，不做复杂扩展。

### 云端部分

- 模型：CLIP `ViT-B/32`
- 数据：ImageNet 训练集或一个较大无标签图像集
- 输出：
  - 图像特征文件 `cloud_features.npy`
  - 样本索引文件 `cloud_meta.json`
  - 检索索引文件 `faiss.index`

### 本地部分

- 模型：ResNet-18
- 数据：ImageNet 子集
- 学习方式：SimCLR 风格对比学习
- 输出：本地编码器权重

### 跨端对齐部分

- 查询：本地编码器输出的特征
- 检索：FAISS top-k 最近邻
- 聚合：按相似度 softmax 加权平均
- 对齐损失：余弦对齐损失

## 3. 具体训练流程

### 阶段 A：构建云端特征库

输入大规模图像数据集，使用 CLIP 图像编码器提取每张图像的特征：

`f_i = normalize(CLIP(x_i))`

然后保存全部特征，并建立 FAISS 索引。

这一阶段只做一次，不参与训练。

### 阶段 B：本地自监督预训练

对每张本地图像生成两个增强视图 `v1` 和 `v2`：

- `v1 = aug(x)`
- `v2 = aug(x)`

送入本地模型：

- `h1 = f_local(v1)`
- `h2 = f_local(v2)`
- `z1 = g(h1)`
- `z2 = g(h2)`

其中：

- `f_local` 是 ResNet-18 编码器
- `g` 是两层 MLP 投影头

本阶段先只训练 SimCLR 损失，让本地特征具备基本可分性。

### 阶段 C：联合训练

在完成本地预训练后，加入云端语义对齐。

对当前样本，用本地特征 `h1` 作为查询，从云端特征库中检索 top-k 邻居：

- `N_k(h1) = {c_1, c_2, ..., c_k}`

计算相似度：

- `s_i = cosine(h1, c_i)`

计算权重：

- `w_i = softmax(s_i / tau_r)`

聚合得到动态语义原型：

- `p = normalize(sum_i w_i * c_i)`

然后同时优化两个目标：

1. 本地自监督一致性
2. 本地特征与云端语义原型对齐

总损失定义为：

`L = L_ssl + lambda_align * L_align`

其中：

- `L_ssl`：SimCLR 的 InfoNCE 损失
- `L_align = 1 - cosine(h1, p)`

如果需要更稳定，可以对 `h2` 也做一次对齐：

`L_align = (1 - cosine(h1, p)) + (1 - cosine(h2, p))`

## 4. 模型与特征空间设计

为了避免检索空间和对比学习空间混在一起，我准备这样做：

- `h`：编码器输出，用于云端检索和语义对齐
- `z`：投影头输出，用于 SimCLR 对比损失

即：

- 检索和对齐在 `h` 空间完成
- 对比学习在 `z` 空间完成

这样更稳定，也更容易调试。

## 5. 第一版固定超参数

第一版先用一组固定参数打通流程：

- 本地模型：ResNet-18
- 云端模型：CLIP ViT-B/32
- batch size：128
- optimizer：AdamW
- learning rate：`1e-3`
- epoch：
  - 预训练 100
  - 联合训练 100
- top-k：10
- `tau_r`：0.07
- `lambda_align`：0.5

这些参数不是最终最优，只是为了先跑通实验。

## 6. 代码实现结构

我准备按下面的结构实现：

### 数据模块

- `datasets/transforms.py`
  - 定义 SimCLR 风格双视图增强
- `datasets/imagenet_subset.py`
  - 读取本地 ImageNet 子集

### 模型模块

- `models/resnet_encoder.py`
  - 封装 ResNet-18 主干
- `models/projection_head.py`
  - SimCLR 投影头
- `models/clip_encoder.py`
  - 云端 CLIP 特征提取接口

### 特征库与检索模块

- `models/feature_bank.py`
  - 保存、加载云端特征库
- `models/retriever.py`
  - FAISS 检索接口

### 训练模块

- `train.py`
  - 训练主循环
  - 预训练模式
  - 联合训练模式
- `main.py`
  - 参数解析和入口

## 7. 我实际要实现的训练顺序

我会按下面顺序推进，而不是一次写完所有东西。

### 第一步：先实现本地 SimCLR 基线

目的：

- 确认数据加载、增强、编码器、投影头、InfoNCE 都正常
- 得到一个不依赖云端的基线结果

### 第二步：实现云端特征库构建脚本

目的：

- 用 CLIP 提取大规模图像特征
- 生成 `.npy + .json + faiss.index`

### 第三步：实现检索与原型聚合

目的：

- 给定本地特征，返回 top-k 邻居和聚合原型

### 第四步：把语义对齐损失接入训练

目的：

- 在 SimCLR 的基础上加入 `L_align`
- 完成完整联合训练

## 8. 评估方式

第一版只做最直接的验证。

### 对比对象

- Baseline：只做本地 SimCLR
- Ours：SimCLR + 云端语义原型对齐

### 评估指标

- 线性分类准确率
- k-NN 分类准确率
- 特征可视化结果

只要 `Ours` 稳定优于 `Baseline`，就说明这个方案值得继续做。

## 9. 当前版本不做的内容

第一版暂时不实现下面这些复杂设计：

- 文本侧 CLIP 语义监督
- 伪标签传播
- 动量编码器
- 多层原型
- 在线更新云端特征库
- 不确定性加权

原因很简单：先验证主链路是否成立。

## 10. 一句话总结

我要实现的不是一般的知识蒸馏，而是一个具体训练框架：

**先用 CLIP 建云端语义库，再用 ResNet-18 做本地自监督训练，并通过 top-k 检索得到的动态语义原型约束本地特征学习。**
